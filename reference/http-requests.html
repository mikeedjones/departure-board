Making HTTP requests from MicroPython

Compressed from Lesson 4.

Raw sockets — what HTTP actually is on the wire

import socket

ai = socket.getaddrinfo("google.com", 80)
addr = ai[0][-1]

s = socket.socket()
s.connect(addr)
s.send(b"GET / HTTP/1.0\r\n\r\n")

print(s.recv(512))

The double \r\n\r\n is the whole protocol: a blank line telling the server "no more headers, this is the end of the request." recv() hands back raw bytes — status line, headers, and body all mixed together, unparsed.

"You can take a low-level approach to HTTP requests using raw sockets, or a high-level approach using the urequests library." Raspberry Pi Ltd, Connecting to the Internet with Raspberry Pi Pico W-series, §3.8

urequests — the same thing, wrapped

import urequests

r = urequests.get("http://httpbin.org/json")
print(r.status_code)
print(r.json())
r.close()
Attribute / methodWhat it gives you
r.status_codeHTTP status as an int (200, 404, ...)
r.contentRaw response body, bytes
r.textResponse body decoded as a string
r.json()Body parsed as JSON — limited support, but covers plain objects/arrays
r.close()Required. See below.
Always call r.close(): per the guide — "if you do not, the object will not be garbage-collected, and if the request is being made inside a loop this will quickly lead to a crash." A departure board polls in a loop forever, so this isn't optional cleanup, it's the difference between running for a day and running for five minutes.

Known limitations

  • HTTPS works, but SSL certificate verification is currently disabled.
  • Redirects are followed automatically.
  • JSON support is limited — fine for plain objects/arrays, don't expect edge-case spec compliance.

Generalizes the lesson-3 status-poll habit: don't just poll at startup, poll again whenever a request fails, and only reconnect if the link actually dropped.

Installing urequests

uv run mpremote mip install urequests

Runs on your computer, not the Pico — mip fetches the package over your machine's network connection, then copies the resulting files onto the device over USB. The device doesn't need its own Wi-Fi connection yet for this step.
