Joining a network in station mode

Compressed from Lesson 3.

The connect sequence

import network

wlan = network.WLAN(network.STA_IF)
wlan.active(True)
wlan.connect(ssid, password)

while max_wait > 0:
    status = wlan.status()
    if status < 0 or status >= 3:
        break
    max_wait -= 1
    time.sleep(1)

if wlan.status() != 3:
    raise RuntimeError("network connection failed")

ip, subnet, gateway, dns = wlan.ifconfig()
"We're using the network library to talk to the wireless hardware." Raspberry Pi Ltd, Connecting to the Internet with Raspberry Pi Pico W-series, §3.6

STA_IF vs. AP_IF

InterfaceRoleUsed here?
network.STA_IFStation — join an existing access point as a clientYes
network.AP_IFAccess point — the board becomes the AP other devices joinNot used in this project

CYW43 link status codes

ValueConstantMeaning
0CYW43_LINK_DOWNNo connection, no activity
1CYW43_LINK_JOINAssociated with the AP, no IP yet
2CYW43_LINK_NOIPJoined, waiting on DHCP for an IP
3CYW43_LINK_UPConnected — has an IP
-1CYW43_LINK_FAILFailed, other reason
-2CYW43_LINK_NONETNo matching access point found
-3CYW43_LINK_BADAUTHWrong password / auth rejected

The values are the CYW43 driver's own enum, passed straight through wlan.status() — the poll loop's status < 0 or status >= 3 check is really "stop looping once we're either definitely done (3) or definitely dead (negative)."

If it won't associate on a known-good network

The wireless country defaults to a conservative worldwide-safe channel set, which can exclude channels your router actually uses:

import rp2
rp2.country('GB')   # ISO 3166-1 alpha-2, set to your own country

vs. the C SDK

The same sequence in C folds country + station-mode + connect-with-timeout into three calls with a blocking timeout parameter, rather than a poll loop you write yourself:

cyw43_arch_init_with_country(CYW43_COUNTRY_UK);
cyw43_arch_enable_sta_mode();
cyw43_arch_wifi_connect_timeout_ms(ssid, pass, CYW43_AUTH_WPA2_AES_PSK, 10000);
