The driver's core primitive

def write_cmd_mpy(self, command, *args):
    self.dc(0); self.cs(0)          # DC low = command
    self.spi.write(bytearray([command]))
    self.cs(1)
    if args: self.write_data(bytearray(args))

def write_data_mpy(self, data):
    self.dc(1); self.cs(0)          # DC high = data
    self.spi.write(data)
    self.cs(1)

Every driver call — clear(), draw_text(), everything — bottoms out here: flip DC, toggle CS, raw spi.write().

vs. Arduino: the DC-toggle trick is identical. What differs is CS is toggled by hand around every single write here — MicroPython's machine.SPI has no beginTransaction()/endTransaction() wrapper.
