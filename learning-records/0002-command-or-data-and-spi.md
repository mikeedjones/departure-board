# Lesson 2 complete: SPI command/data framing, confirmed on real hardware

Wired the ILI9341 breakout over 4-line SPI and hit the win condition (navy screen, white "Departure Board" text) — confirmed the DC pin as an ILI9341-specific escape hatch for command/data framing, not a generic SPI signal, and correctly reasoned (via quiz) that the breakout has no MISO/SDO because the driver only ever writes, never reads back from the display. Also absorbed the vs.-Arduino contrast unprompted: `machine.SPI` is a thinner abstraction than Arduino's SPI library, with CS toggled by hand in plain Python around every transaction rather than via a `beginTransaction()`/`endTransaction()` wrapper. Display is now proven working independent of network state — safe to treat display work and the upcoming Wi-Fi track as separable, interleavable threads rather than one blocking the other.
