What's actually in the box, before you wire anything

Before touching a breadboard — not the point of this lesson, you've done that plenty — it's worth knowing what's actually inside the two active components in this kit, and specifically what changed in the newer silicon you're holding versus the boards most tutorials online still assume.

The win, in one sentence
You'll know what's silicon-level different about the "2 W" in your board's name, be able to find the wireless chip on the pinout diagram, and know how it talks to the main chip — which turns out to matter well before you write a line of Wi-Fi code.

What's actually on the Pico 2 W

The board is built around the RP2350 — a genuinely different chip from the RP2040 that older "Pico W" boards and most existing tutorials use, not just a speed bump:

"Dual Cortex-M33 or RISC-V Hazard3 cores clocked at up to 150MHz... 520 kB multi-bank high performance SRAM... External Quad-SPI flash with eXecute In Place (XIP) and 16kB on-chip cache... RP2350 microcontroller with 4 MB of flash memory." Raspberry Pi Pico 2 W Datasheet, Ch.1 "About Pico 2 W"

The "Cortex-M33 or RISC-V Hazard3" part is the interesting bit: the RP2350 can run either instruction set, selectable at boot. MicroPython on Pico 2 W runs the Arm cores — the RISC-V option matters mainly if you ever drop to C/the Pico SDK, which per the mission you're deliberately not doing here. Worth knowing it exists; not worth chasing further right now.

The wireless chip — and why it already matters

The "W" is a separate chip, not a feature baked into the RP2350 itself:

"Pico 2 W has an on-board 2.4GHz wireless interface using an Infineon CYW43439... The wireless interface is connected via SPI to the RP2350." Raspberry Pi Pico 2 W Datasheet, Ch.1 "About Pico 2 W"

That's not a coincidence worth skipping past: the same bus you just traced command-by-command in Lesson 2 — CS, SCK, a data line, one chip talking to another — is exactly how the RP2350 talks to its own Wi-Fi radio internally. When the mission gets to fetching live departure data, network.WLAN will look like a totally different API on the surface, but underneath it's issuing SPI transactions to the CYW43439 the same shape as write_cmd_mpy/write_data_mpy issues to the display. You won't be starting from zero a second time.

Board name check: a plain "Pico 2" (no W) is the same RP2350 chip with no CYW43439 and no wireless — cheaper, but a dead end for this mission. Worth a glance at the board silkscreen to confirm you actually have the W variant before going further.
